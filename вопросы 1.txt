Система контроля версий разрешает нескольким участникам (контрибуторам) одного проекта свободно перемещаться между всеми ветками, и копировать код из других веток. Иначе
постоянно приходилось бы ждать обновления мастер-ветки.

2)
clone - копировать данные с гитхаба.
push - добавление данных на гитхаб.
commit - добавить файл в локальный репазиторий.
add - добавление содержимого файла в индекс.
branch - создание, удаление новых веток.
checkout - просмотрение содержимого различных веток
merge - слияние веток, в некоторых случаях открывает меню merge, где мы пытаемся устранить несостыковки файлов.
(я не уверен нужен ли init и origin )

3)
Существуют Гомогеные и Гетерогеные контейнерыю. Гомогеные контейнеры могут содержать только один тип данных, гетерогеные могут содержат несколько типов;
Типы гомогеных контейнеров:
1. Основаны на массиве (Элементы лежат по порядку) - vector, dequeue, array
2. Основаны на дереве - set, multiset, map (отсированы, но лежат иначе чем в предыдущем)
3. Узловые - list 
Типы гетерогеных:
1. Хэш-таблица (несортированные, работают быстрее)
2. адаптеры: stack-lifo, queue-fifo, priority_queue.
